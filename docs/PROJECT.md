# Projeto

## Principios de desenvolvimento

O ANATEL Dashboard sera desenvolvido em sprints pequenas, sempre com entregas compreensiveis e revisaveis.

O projeto deve evitar complexidade desnecessaria. Cada decisao tecnica deve ajudar o produto a ser mais simples, rapido, confiavel ou facil de manter.

## Regras gerais

- Nao implementar muitas funcionalidades de uma unica vez.
- Priorizar codigo claro e modular quando o codigo comecar a ser criado.
- Documentar decisoes importantes.
- Evitar dependencias desnecessarias.
- Evitar telas e fluxos que deixem o uso confuso.
- Manter o usuario protegido de detalhes tecnicos sempre que possivel.

## Forma de trabalho por sprint

Cada sprint deve conter:

- Objetivo.
- Arquivos alterados.
- Resumo tecnico.
- Testes realizados.
- Proximos passos.

Ao terminar uma sprint, o trabalho deve parar para revisao antes da proxima etapa.

## Perfil esperado do sistema

O sistema deve parecer uma ferramenta de trabalho: direto, limpo, rapido e previsivel.

O usuario principal nao precisa ser programador. Por isso, operacoes importantes como atualizar a base devem ser guiadas por botoes, mensagens claras e feedback visual.

## Criterio de qualidade

Uma funcionalidade so deve ser considerada pronta quando:

- resolve o problema real;
- e facil de usar;
- tem comportamento previsivel;
- nao torna o projeto pesado sem necessidade;
- pode ser alterada no futuro sem reescrever tudo.
